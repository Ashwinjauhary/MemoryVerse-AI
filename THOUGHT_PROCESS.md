# Thought Process & Reflection

## Problem Understanding
As a student, I realized that our digital footprints are incredibly scattered. We constantly accumulate certificates, resumes, event passes, and project reports, but they end up buried across different Google Drive folders, local hard drives, and emails. When it's time to build a portfolio or look back at a specific timeline of achievements, there is no easy way to see the connections between these files. I wanted to build a system that acts as a "second brain" specifically for these personal milestones, transforming a messy folder hierarchy into a unified, spatial knowledge repository.

## Approach
Rather than building a standard folder or tag-based organization system—which still requires heavy manual effort from the user—I centered the solution on semantic embeddings and LLM-driven relationship mapping. This elevates the app from a simple storage locker to a genuine AI application. By mathematically converting documents into vector embeddings and using semantic search to find connections, the system constructs a dynamic knowledge graph automatically. It fundamentally shifts the burden of organization from the human to the AI.

## Architecture Rationale
Because I built this within a tight hackathon-style timeframe, I needed an architecture that was extremely fast to iterate on but robust enough to support complex AI workflows. I chose Next.js for the frontend to quickly spin up a beautiful, responsive UI, and FastAPI for the backend because Python is simply unmatched for integrating ML libraries like PyMuPDF and sentence-transformers. Finally, Supabase with the `pgvector` extension was the perfect database choice because it allowed me to store standard relational metadata right next to my vector embeddings, eliminating the headache of syncing two separate databases.

## Challenges Faced
The most frustrating challenge I faced was OCR reliability on scanned certificates, as complex fonts or seals often confused Tesseract, requiring me to implement robust error handling in the pipeline. I also wrestled with malformed JSON responses from the LLM during categorization; I had to implement retry logic and strict system prompts to ensure the data could be safely parsed by my backend. Additionally, tuning the relationship-confidence threshold was highly iterative—if I set it too low, the graph became noisy with irrelevant connections, but if I set it too high, it missed obvious links. Finally, implementing secure JWT authentication to ensure strict data isolation between different users required careful middleware configuration.

## Tradeoffs
To optimize for speed and cost, I made several calculated tradeoffs. I chose to generate vector embeddings locally using HuggingFace's `MiniLM` rather than an API-based model like OpenAI's `text-embedding-3`. While the API would yield slightly higher dimensional accuracy, running the model locally is entirely free, reduces network latency, and is perfectly sufficient for personal document retrieval. Similarly, using an LLM to categorize documents is highly flexible and handles edge-case files beautifully, but it is undoubtedly slower and more computationally expensive than using a rigid, rule-based classification algorithm.

## What I'd Do With More Time
If I had more time to develop MemoryVerse, I would train a small, fine-tuned classification model to replace the LLM categorization step, vastly speeding up the ingestion pipeline. I would also implement multi-modal embeddings capable of understanding the visual elements of a certificate (like university logos or official seals) rather than relying purely on extracted text. Additionally, I would love to build a career-path recommendation feature on top of the relationship graph, suggesting potential job roles based on a user's clustered skills and achievements. Finally, I would migrate the frontend route protection from client-side state logic to secure server-side Next.js middleware.

## Success Metric Reflection
The ultimate test for MemoryVerse was achieving that "I never have to search through folders again" feeling, and I believe the current semantic search and timeline implementation successfully delivers that demo moment. Being able to type "that hackathon where I used React" and instantly see the certificate without remembering its filename is incredibly satisfying. To make it land even better in a live demo, I'd eventually want to render the relationships as an interactive, visual 3D node graph so users can literally see the web of their achievements spinning out in real time.
